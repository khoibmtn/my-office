# /brainstorm - Brainstorming Protocol

## Purpose
Use this skill when you are asked to implement a new feature or complex architecture but the requirements are vague or underspecified. Do NOT guess the user's intent.

## Protocol: Socratic Gate
1. **STOP** - Do NOT start coding.
2. **ASK** - Formulate 3 specific, domain-relevant questions to clarify:
   - 🎯 **Purpose**: What is the core problem being solved?
   - 👥 **Users**: Who is the target audience for this feature?
   - 📦 **Scope**: What are the absolute must-haves vs. nice-to-haves?
3. **WAIT** - Stop execution and wait for the user's response before proceeding to write an implementation plan.

## Question Format
When asking questions, present options with trade-offs rather than open-ended text fields.
For example:
"Do you want the date filter to apply to (A) Issue Date only, (B) Completion Date only, or (C) Both? Option A is faster to implement, but C gives more flexibility."
